# 02. 아키텍처 & 의사결정 기록 (ADR)

구현 중 "왜 이렇게 되어 있지?"가 나오면 여기를 먼저 본다. 결정을 뒤집으려면 해당 ADR의 근거를 반박해야 한다.

## 아키텍처 전경
```
Streamlit UI
   │ (진행 상황 스트리밍)
LangGraph 상태 그래프
   [교정 L1~L3] → [분해+분석 LLM 1콜] → [계획 검증 8종] → [그라운딩]
   → 실행기(ready-set 루프) { resolve_hop = 검색(A벡터+B그래프) → Quad 검증 → 실패 시 힌트 재질의 }
   → [final_op(결정적 비교)] → [Path-Check] → [응답 조립]
   │
   ├─ ChromaDB: 문단 벡터 인덱스
   ├─ NetworkX: 지식그래프 (GraphPort 뒤) + alias 사전
   └─ JSON 구조화 로그 (request_id 관통)
```

## ADR-1. 검토를 LLM judge가 아닌 그래프 조회로
- 문제: 오답 생성 모델과 심사 모델이 같으면 그럴듯한 오답이 통과. 탈락 사유도 안 남음.
- 결정: type/name/backlink는 그래프·사전 조회(결정적, LLM 0콜), desc만 임베딩 유사도.
- 근거: EMNLP'25 서베이 분류상 KG의 "Validator" 역할 — 덜 개척된 포지션. 발표 차별화의 본체.

## ADR-2. 질문 표현 = hop 원자 + start 참조 (유형 분기 없음)
- 여정: 3유형 분류 → 인지과학 5축(Lehnert/Graesser) → 데이터셋 기준 감량 → CHAIN/STAR/PARALLEL 위상 3형 → **"모든 형태에서 relation·type 비교는 같고 시작점만 다르다"**는 통찰로 통일.
- 결정: hop = (start, relation, direction, expected{type,definition}, sub_query). start가 리터럴이면 앵커, `{n}`이면 hop n의 답 참조. 직렬/팬아웃/병렬/혼합이 참조만으로 표현됨. 답 = 아무도 참조하지 않는 leaf (final_op 있으면 그 출력).
- 이론 근거: QDMR(Break, TACL 2020)의 SELECT/PROJECT/#참조와 동형. 13개 연산자 중 결정적 비교만 final_op로 지원, FILTER/AGGREGATE류는 스코프 아웃.

## ADR-3. 실행 = 개념은 재귀, 구현은 ready-set 반복
- 결정: resolve(start, relation)→t 를 공통 원자로, 의존성 해소된 hop부터 반복 실행 (asyncio.gather).
- 재귀 함수 구현 금지 이유: 팬아웃/분기 시 병렬성 상실, 재시도·부분실패가 콜스택에 얽혀 trace 불가.
- 부수 효과: 병렬성 공짜, 유형별 실행기 코드 소멸.

## ADR-4. resolve 단일화가 여는 재사용 최적화 (설계에 내장)
- 메모이제이션: **[4] 경로A 검색결과만 캐시** (키=(start_resolved, relation_grounded, direction, sub_query, definition)). 검증·재질의는 매번 재실행(quad=0콜이라 비용손실 없음). 검증결과·verifier·hop_id·retries·FAILED를 캐시에 박으면 3단 비교/재평가가 무증상 오염되므로 금지 (C6). **키에 direction 누락 금지.**
- verifier 주입: resolve(verifier=quad|llm_judge)로 **Ours−G ablation arm**(FULL 파이프라인 + llm_judge)을 공짜 구현 → "이득이 분해가 아닌 검증 방식"임을 분리 증명. (Agent-basic은 분해가 없어 이 구조로 못 만듦 — 별도 현실 대조군, 05 §1/C4.)
- 앵커 호이스팅·배칭·프롬프트 캐싱: Day 11+ 튜닝 항목.
- 메모이제이션과 verifier 주입은 나중에 붙이기 어려우므로 처음부터.

## ADR-5. 그래프 저장 = NetworkX (Neo4j 아님)
- 근거: 연산 전량이 1-hop 이웃/엣지 존재/속성/제약 BFS — 인메모리 O(1)급. 노드 수만 규모, 단일 프로세스. Neo4j 우위 조건(램 초과·패턴쿼리·동시접근) 해당 없음. 2주에 인프라 리스크만 추가.
- 안전장치: GraphPort(Protocol)로 감싸 교체 가능. Chroma는 문단 벡터 전담(이웃 연산 안 함 — 역할 혼동 금지).

## ADR-6. relation 그라운딩 = 무학습 런타임 정렬 (RoG 파인튜닝 대체)
- 문제: RoG는 KG 어휘를 파인튜닝으로 증류 → KG마다 재학습. ToG는 홉마다 prune LLM 콜 → 비쌈.
- 결정: 분해가 relation을 먼저 뱉으므로(계획 선행) anchor 노드의 실존 엣지 집합에 정렬만: exact → 임베딩 스냅(>θ_high) → LLM 타이브레이크(θ_low~θ_high, 객관식 1콜) → 포기(<θ_low, 실존 relation 목록을 재질의 힌트로).
- 함정과 방어: bornIn↔diedIn은 임베딩상 가까움 → 이중 임계값으로 타이브레이크 유도 + **backlink 검증은 grounded가 아닌 분해가 의도한 원본 relation 기준**이라 스냅 오염이 회수됨 (그물 2겹).

## ADR-7. 교정 = 4계층, LLM 자유 교정 금지
- L1 규칙(NFC·공백·약어) → L2 alias 대조(정상 질문 조기 종료) → L3 편집거리+자모거리, **후보를 그래프 실존 엔티티로 제한**(과교정 구조적 차단) → L4 분해 콜 통합 객관식(후보 중 선택 or 원문 유지, diff 강제).
- alias 사전(위키 redirect)이 교정·링킹·name검증 3역 겸용.

## ADR-8. hop 다답 = top-1 기본 + margin 조건부 분기, 도입은 측정 후
- 문제: top-1 greedy는 과거 hop 오선택의 복구 경로가 없음(재질의는 현재 hop만 고침). SOTA도 복수 경로 유지(ToG 빔, RoG top-3).
- 결정: 기본 top-1, margin<ε 또는 저신뢰 시 top-2 분기, 활성 체인 ≤3. 체인 간 승자 = 검증 점수 합 + Path-Check 중재.
- 도입 판정: Day 8에 candidates 전량 로그로 oracle@1 vs @2 실측 → 갭 크면 구현(반나절), 작으면 top-1 유지. **candidates를 top-1만 남기고 버리면 이 분석이 불가능해짐 — 전량 보존.**
- 부수 효과: 동명이인 해소가 이 메커니즘으로 해결.

## ADR-9. 분해 메타데이터 확장 + 정답 누출 방지
- 결정: hop에 expected.definition("베토벤이 태어난 도시" — 조건의 재서술)과 anchor_meta{name, disambiguator} 추가.
- 용도: desc 검증을 "후보 설명문 vs 기대 정의" 비교로 정밀화, sub_query+definition 결합 임베딩으로 검색 보강(HyDE 패턴), disambiguator로 앵커 링킹.
- **금지:** definition에 답이 될 고유명사 포함 금지. 위반 시 LLM 사전지식이 검증을 오염(맞으면 무의미, 틀리면 오답 강화). 프롬프트 규칙 + 코드 검사(계획 검증 8번: definition 내 alias 엔티티명 검출 시 재생성).

## ADR-10. 4계층 + ports 5개 (core/infra 2분할 → 클린 아키텍처)
- 배경: 이 프로젝트는 1인 개발이지만 구현을 병렬 멀티에이전트로 진행한다. 에이전트는 전체 설계를 머릿속에 들고 있지 않고, 편하면 계층 경계를 넘고(usecase에서 adapter 직접 호출), 방치하면 레포지토리·팩토리·CQRS를 습관적으로 들여온다. 사람 1명이면 과설계인 명시적 경계가, 병렬 에이전트에겐 값싼 보험이 된다.
- 결정: domain ← usecases ← adapters ← bootstrap/apps/scripts 4계층. 의존은 화살표 방향만. seam은 ports.py의 Protocol 5개(LLMPort/EmbedderPort/VectorIndexPort/GraphPort/TracerPort)로 고정하고, usecase는 이 5개만 본다(구체 구현의 존재를 모른다).
- 이전 결정 뒤집기: ADR-5의 "Protocol은 GraphStore 하나만(과설계 금지)"을 5개로 확대. 근거 반박이 아니라 전제 변화다 — GraphStore 1개 결정은 "런타임 다형성"을 기준으로 삼았고(구현 1개면 Protocol 불필요), ADR-10은 "에이전트가 구현을 안 읽고 계약만 읽는 격리"를 기준으로 삼는다. 파이썬 테스트는 덕타이핑으로 Protocol 없이도 가능하지만, ports.py 한 파일이 모든 seam을 열거하면 에이전트의 컨텍스트 지도가 된다.
- 상한과 방어: **포트 5개가 천장. 6번째를 추가하려면 이 문서에 ADR을 먼저 쓴다.** 에이전트는 반드시 포트를 늘리려 하므로 이 천장이 규율의 핵심이다. 계층 위반은 tools/check_layers.sh(grep 게이트)가 자동 검출 — 커밋 전 통과가 done 기준.
- 함정: 계약을 산문으로만 적으면 에이전트 A와 B가 같은 모듈을 다르게 구현한다. 에이전트는 문장은 무시하고 실패하는 테스트는 따른다. 그래서 06의 모듈 계약표는 시그니처(models.py 타입)·import 허용목록·테스트 파일까지 실행 가능한 형태로 못박는다.
