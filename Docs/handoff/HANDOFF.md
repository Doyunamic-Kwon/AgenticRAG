# VeriHop 개발 핸드오프 (Claude Code 시작점)

## 30초 요약
한국어 복합 질문("모차르트가 태어난 도시의 대표 축제는?")을 hop 단위로 분해하고,
각 hop의 검색 결과를 **경량 지식그래프 기반 4중 검증**(type/name/desc/backlink)으로 걸러내며,
실패 시 그래프 정보를 힌트로 질의를 재작성해 재검색하는 **Agentic RAG** 시스템.
가천 AI 부트캠프 프로젝트 12번 과제. 기간 14일. 로컬 실행 데모가 최종 산출물.

핵심 차별화: 검색 결과 검토를 LLM judge(확률적)가 아니라 **그래프 조회(결정적)**로 수행.
검증 4종 중 3종이 LLM 호출 없이 동작한다.

## 절대 원칙 (전 작업 공통)
- 프로젝트 제목이 "검색 성능 향상"이다. 모든 판단 기준은 **검색 품질 지표**.
- Day 2(Baseline 수치), Day 6(시나리오 B 재현), Day 8(Ours > Agent-basic) 게이트는 협상 불가.
- Day 8 미달 시 데모 폴리싱을 버리고 검증 개선에 투입. 데모가 예뻐도 숫자 없으면 실패.
- 문서와 코드가 충돌하면: `03_파이프라인_명세.md` > 나머지 문서. 명세 변경 시 명세부터 고친다.
- LLM 콜 예산: 2-hop 정상 경로 4콜 (분해1 + hop후보추출2 + 답변1). 늘리는 변경은 근거 필요.
