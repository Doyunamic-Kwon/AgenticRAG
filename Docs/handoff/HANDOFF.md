# VeriHop 개발 핸드오프 (Claude Code 시작점)

## 30초 요약
한국어 복합 질문(예: "모차르트가 태어난 도시의 대표 축제는?")을 hop 단위로 분해하고, 각 hop의 검색 결과를 경량 지식그래프 기반 4중 검증(type/name/desc/backlink)으로 걸러낸다. 검증에 실패하면 그래프 정보를 힌트로 질의를 재작성해 다시 검색한다. 이런 방식의 Agentic RAG 시스템이다. 가천 AI 부트캠프 프로젝트 12번 과제이고, 기간은 14일, 로컬 실행 데모가 최종 산출물이다.

이 시스템의 차별점은 검색 결과 검토를 LLM judge(확률적)가 아니라 **그래프 조회(결정적)**로 수행한다는 데 있다. 검증 4종 중 3종은 LLM 호출 없이 동작한다.

## 즉시 착수 순서
1. `07_첫번째_태스크.md`의 W0 스크립트를 구현하고 실행한다. 미검증 가정 2개를 사실로 만드는 작업이다.
2. W0 결과로 `configs/relations.yaml`의 relation 10종을 확정한다.
3. 이후 `04_WBS.md`의 Day 1부터 진행한다.

## 절대 원칙 (전 작업 공통)
- 프로젝트 제목이 "검색 성능 향상"이다. 모든 판단 기준은 **검색 품질 지표**로 잡는다.
- Day 2(Baseline 수치), Day 6(시나리오 B 재현), Day 8(Ours > Agent-basic) 게이트는 협상하지 않는다.
- Day 8에 미달하면 데모 폴리싱을 버리고 검증 개선에 투입한다. 데모가 예뻐도 숫자가 없으면 실패다.
- 문서와 코드가 충돌하면 `03_파이프라인_명세.md`를 나머지 문서보다 우선한다. 명세를 바꿀 때는 명세부터 고친다.
- LLM 콜 예산은 2-hop 정상 경로 기준 4콜이다(분해 1 + hop 후보 추출 2 + 답변 1). 이를 늘리는 변경에는 근거가 필요하다.
