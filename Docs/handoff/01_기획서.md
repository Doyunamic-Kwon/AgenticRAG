# 01. 기획서 (개발 기준판)

## 1. 과제 요구사항 (외부 제약 — 변경 불가)
브리프(프로젝트 12번) 필수 루프: 질의 계획 → 검색 → 검색 문서 검토(관련성 평가) → 부족 시 질의 교정·재정규화 → 재검색.
데모 필수 표시: 세운 계획, 재작성한 질의, 반복 횟수, 선택한 근거, 출처.
권장 스택: Python, ChromaDB, text-embedding-3-small, gpt-4o-mini, LangGraph(또는 직접 상태 루프), Streamlit.

| ID | 브리프 요구 | 구현 |
|---|---|---|
| R1 | 질의 계획 | 질문 분해(hop 그래프) + ready-set 실행 계획 |
| R2 | 검색 | Chroma 벡터 + 그래프 이웃 이중 경로 |
| R3 | 검토 | Quad Verification (차별화 본체) |
| R4 | 교정·재정규화 | 4계층 교정 + 그래프 힌트 재질의 |
| R5 | 재검색 루프 | hop당 max 2회 + "확인 불가" 탈출 |
| R6 | 과정 노출 데모 | Streamlit hop trace + 검증 뱃지 |
| R7 | 검색 성능 측정 | 3단 비교 (Recall@k/MRR/nDCG) |

## 2. 해결하는 문제
1. **복합 질문 검색 실패** — 질문 전체 1회 검색은 hop별 필요 문서가 섞여 정밀도 붕괴
2. **중간 오류 무검증 전파** — 1홉 오답이 그대로 진행되어 자신 있는 최종 오답 (최우선 문제)
3. **검토 불투명성** — LLM judge는 통과/탈락의 구조적 근거를 남기지 않음

## 3. 핵심 가치 (단 하나)
"틀린 중간 결과를 답이 되기 전에 걸러내고, 걸러낸 이유를 보여준다."
속도·UI보다 검증 실패→재질의→회복 과정의 정량적·시각적 증명이 우선.

## 4. Agent 자율성 범위
- 자율: 분해 방식, 검색 경로, 후보 채택/기각, 재질의 여부·내용, 지름길 감지 시 잔여 hop 스킵
- 정지: 재질의 2회 소진 시 추측 금지("확인 불가"), 분해 검증 실패 시 SIMPLE 강등, 주관 비교 거부
- 원칙: **자율성 상한 = 검증 가능한 범위**

## 5. 데이터 전략
- 코퍼스: 한국어 위키. 단, "인물 카테고리 크롤링"이 아니라 **평가 문항의 원문 문서 + 하이퍼링크 이웃으로 역정의** (코퍼스-평가셋 정합 보장. 정답 문서가 코퍼스에 없으면 Recall 측정이 무의미해지므로)
- 단일홉 평가: KorQuAD (해당 문서 범위 필터)
- 멀티홉 평가: **자체 구축 100~200문항** — KorQuAD QA 쌍 브릿지(answer(QA1)==entity(QA2)) 방식. 문항마다 중간 정답·근거 문단·relation 사슬 기록 필수 (05_평가_명세 참조)
- 라이선스: KorQuAD 1.0 = CC BY-ND → 변형 데이터 재배포 금지, 재생성 스크립트만 공개
- 순환 평가 금지: 평가셋 원천(KorQuAD)이 시스템(자체 추출 그래프)과 독립이어야 함. 그래프 경로에서 문항을 역생성하지 말 것

## 6. KPI
- 1차 기준: 멀티홉 최종 Recall@10이 **Agent-basic 대비 +X%p(측정 후 확정)** + Ours 절대치. (Baseline 대비 수치는 shortcut 필터 구성상 자동 달성이라 헤드라인 아님 — 05 §4/C2). 추가로 Ours > Ours−G로 검증 기여 분리 증명
- 보조: 중간 정답률, 오류 전파 차단율, 재질의 성공 전환율, 질문당 콜 수·비용·latency

## 7. MVP 경계
- 필수: R1~R7 전부, 직렬(CHAIN) 완전 지원, 시나리오 A/B/C 재현, 3단 비교 수치, 로컬 실행
- 여유 시: STAR(팬아웃), 결정적 비교(final_op), 나란히 비교 UI
- 제외: 주관 비교(MORE_FAMOUS), FILTER/AGGREGATE/SUPERLATIVE, 4-hop+, 역질문 대화, 전면 빔서치, 대화 메모리, 배포·동시성

## 8. 데모 시나리오 (개발 완료 판정 기준이기도 함)
- A (기본): "베토벤이 태어난 도시가 속한 나라는?" → 분해 표시 → hop별 검증 뱃지 → "독일" + 근거
- B (하이라이트): "모차르트가 태어난 도시의 대표 축제는?" → 후보 '빈' backlink 탈락 표시 → 힌트 재질의 → '잘츠부르크' 확정 → 진행. LLM judge라면 '빈' 통과 가능한 케이스임을 비교 모드로 시연
- C (오타): "베토밴 교향곡 9번 초연 장소" → L3 교정 → SIMPLE 판정 → 즉답
