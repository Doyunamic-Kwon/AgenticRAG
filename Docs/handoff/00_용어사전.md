# 00. 용어사전 (표기 통일)

발표와 문서 전반의 용어·표기를 여기서 고정한다. 새 용어를 도입하면 여기에 먼저 추가한다.

## 표기 원칙

- 핵심 기술 용어는 영문을 유지한다: hop, relation, backlink, verifier, alias, corpus, oracle, Recall/MRR/nDCG, HyDE, direction(fwd/inv).
- 서술은 한글로 쓰고, 코드·식별자·스키마 필드는 원문 그대로 둔다(resolve_hop, GraphPort, verify_rules, θ_high 등).
- 음차가 정착한 용어는 한글로 쓴다: 그라운딩, 앵커, 메모이제이션.

## 핵심 용어

| 용어 | 표기 | 뜻 |
|---|---|---|
| hop | hop (합성어만 멀티홉/단일홉) | 복합 질문을 나눈 한 단계. (start, relation, direction, expected, sub_query) |
| 질문 | 질문 | 사용자의 자연어 입력 (raw_question) |
| 질의 | 질의 | 검색기에 던지는 쿼리 (sub_query, 재질의) |
| 앵커 | 앵커 (anchor) | hop의 시작이 리터럴 엔티티일 때. `{n}`이면 hop n의 답을 참조 |
| relation | relation | 엔티티 간 관계. 스키마 10종 + relatedTo |
| direction | fwd / inv | relation 방향 |
| 그라운딩 | 그라운딩 | 분해가 낸 relation을 그래프 실존 엣지에 맞추는 [3] 단계 |
| Quad Verification | Quad 검증 | type / name / desc / backlink 4중 검증 |
| backlink | backlink | 후보→앵커 역엣지 존재 검사. pass / absent / contradict 3값 (C1) |
| verifier | verifier | resolve_hop에 주입하는 검증 함수. quad 또는 llm_judge |
| 재질의 | 재질의 | 검증 실패 시 힌트를 넣어 질의를 다시 만드는 [6] 단계 |
| tie | tie | top-1과 top-2의 margin<ε. 체인 분기 |
| final_op | final_op | 결정적 비교 연산 (EARLIER/LATER/SAME/GREATER/SMALLER) |
| mode | FULL / SIMPLE | 파이프라인 실행 모드 (PipelineState.mode) |
| 평가 모드 | Baseline / Agent-basic / Ours−G / Ours | 4모드 비교 (05 §1) |
| gold / 예측 | gold / 예측 | 평가셋(불변) 대 런 출력(런마다 생성) |

## 혼용 금지 (왼쪽으로 통일)

| 통일 표기 | 쓰지 않을 것 | 비고 |
|---|---|---|
| hop | 홉 | 단, 멀티홉·단일홉 합성어는 유지 |
| 질의 | 쿼리 | 질문(사용자 입력)과 질의(검색 쿼리)는 다른 개념이니 구분 |
| 검증 | 검토 | 검토는 브리프 R3를 인용할 때만(관련성 평가) |
| 재질의 | (혼용) 재검색·재작성 | 재검색은 R5 루프, 재작성은 브리프 R6 인용에만 |
| 후보 | (서술의) candidate | 코드·스키마 필드는 Candidate |
| llm_judge | (단독) judge | 서술에서 "LLM judge"는 허용 |
| 분해 | (서술의) decompose | 코드는 decompose |
